## What's the Difference?

### Skills vs Agents vs Recipes

Think of it like a kitchen:

- A **skill** is like a recipe card — written instructions for ONE task (e.g., "how to review code"). It's a markdown file that tells your AI what to do.
- An **agent** is like a team member — a chef who knows which recipe cards to grab and when. It has a role, a personality, and a set of skills.
- A **recipe** (cookbook) is a finished dish with the full code — copy-paste and run right now. No setup required.

> 💡 **Key insight:** You can use recipes without skills or agents. They're completely independent.

### auto-research vs ralph-loop

**auto-research** — Your AI tries changes to your code, runs tests, keeps improvements, discards failures.
Like a tireless intern running experiments overnight.

- Best for: optimization, performance tuning, "make this faster."
- Example: "Try 20 different caching strategies and keep the fastest one."

**ralph-loop** — Your AI picks a task from a list, implements it, validates it, commits, moves to the next.
Like a junior developer working through a TODO list.

- Best for: knocking out a backlog of small tasks.
- Example: "Here are 15 small bug fixes. Do them all, one by one."

### knowledge-wiki vs forge-memory

**knowledge-wiki** — A personal Wikipedia. You drop in articles, notes, and docs. The AI organizes them into a searchable wiki with cross-references.

- For: research, learning, business intelligence.
- You set it up yourself. You feed it content.

**forge-memory** — CopilotForge's built-in memory. It remembers your project decisions and coding conventions so the wizard gets smarter over time.

- For: project setup. Created automatically by the wizard.
- You don't manage it — it manages itself.

> 💡 **These are different things.** You don't pick one over the other. knowledge-wiki is a tool you use; forge-memory is a system that runs in the background.

### FORGE.md vs README.md vs SKILL.md

| File | What it is | Who reads it |
|------|-----------|-------------|
| **FORGE.md** | Your project's control panel (generated by the wizard) | You — to see what's set up |
| **README.md** | The repo's welcome page | Everyone — the front door |
| **SKILL.md** | Instructions for an AI assistant (lives in `.github/skills/`) | Your AI — tells it *how* to do a job |

## You DON'T Need...

This is your permission slip to skip things. Seriously.

- ❌ **You don't need agents** if you just want code recipes → go straight to `cookbook/`
- ❌ **You don't need memory** if this is a one-time project → say "no" to Question 3 in the wizard
- ❌ **You don't need the cookbook** if you just want AI-assisted development → the wizard + skills are enough
- ❌ **You don't need to understand skills** to use CopilotForge → the wizard creates them for you
- ❌ **You don't need ALL 24 recipes** → the wizard picks the right ones for your stack
- ❌ **You don't need npm or Node.js** to use CopilotForge → manual copy works fine (only recipes need a runtime)
- ❌ **You don't need to read every doc** → this page points you to the one you need

## Still Not Sure?

Three safe starting points — pick any one:

1. **Just run the wizard.** Type `npx @bradliebs/copilotforge init` and answer the questions. It figures everything out.
2. **Try the simplest recipe.** Open `cookbook/hello-world.ts` (or `.py`). Copy it. Run it. See what happens.
3. **Ask your AI assistant.** Paste this into your AI chat: *"Read FORGE.md and help me get started with CopilotForge."*

Any of these gets you unstuck in under 5 minutes.
